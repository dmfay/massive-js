Module: where

module:where

Query conditions generator.
Parameters:
Name Type Attributes Default Description
conditions Object <optional>
 {} Query criteria mapping column names (optionally including operation eg 'my_field <>') to the parameter values. Predicates generated from a criteria object are joined together with `and`; an `or` key denotes an array of nested criteria objects, the collected predicates from each of which are parenthesized and joined with `or`.
offset Number <optional>
 0 Added to the token index value in the prepared statement (with offset 0, parameters will start $1, $2, $3).
prefix String <optional>
 '\nWHERE ' SQL snippet prefixed to the generated predicate string.
generator String <optional>
 generator Generator functions mutate the SQL output.
Source:
  • lib/query/where.js, line 265
Returns:
An object representing the query conditions. The `conditions` field contains the final SQL string, the `params` field the parameters for the prepared statement, and the `predicates` field the raw predicate mapping.
